# Historique des grandes décisions — Culture

> Repères chronologiques de haut niveau sur les décisions structurantes du projet. **Ce n'est pas un journal détaillé** : pour le détail exact d'une évolution (avant/après, itérations intermédiaires), `git log`/`git blame` sur les fichiers concernés font foi. Les patterns de code qui ont émergé de ces décisions vivent dans `.claude/rules/` (chargés automatiquement quand pertinent) ; l'état actuel du produit vit dans `docs/product-spec.md`. Pas chargé automatiquement — à lire seulement pour comprendre le contexte historique d'une zone avant de la modifier en profondeur.
>
> Dernière mise à jour : 12/07/2026

- **31/05/2026 — Migration de marque Evalia → Culture.** Thème violet → vert/crème, nouveau logo (germe/arbre), typographie Inter Tight × Caveat. Terminée.

- **13–14/06/2026 — Fusion Dashboard/Recherche + refonte Paramètres d'atelier.** `/dashboard` devient la page unique « mes ateliers + recherche + Preview » (`/search` redirige dessus). Tags d'atelier (8 caractères), couverture personnalisée (image ou dégradé), page Paramètres réorganisée en 4 sections indépendantes (Général / Membres & rôles / Briques / Atelier Premium). Introduction du pattern « modifications non enregistrées » (voir `.claude/rules/frontend-patterns.md`) et du modèle Atelier Premium avec irréversibilité garantie par trigger DB (voir `.claude/rules/server-architecture.md`).

- **14–21/06/2026 — Construction du générateur d'examen.** `ExamenTab.tsx`/`QuestionEditor.tsx` : banque de questions, éditeur de question multi-parties, liaison puis suppression du système de liaison au profit de parties, aperçu A4 paginé en direct, brouillon persistant. De nombreuses itérations UI ont eu lieu sur cette période (système de sélection, bandeau, tri/filtres) — l'état stable est documenté dans `docs/product-spec.md` § Générateur d'examen ; les pièges React/CSS rencontrés (mesure de hauteur, `position:fixed` dans un ancêtre transformé, timing de `setState`) sont dans `.claude/rules/frontend-patterns.md`.

- **21–22/06/2026 — Passe de sécurité + modèle de rôles.** Introduction du rôle `manager` (entre `owner` et `member`), centralisation du contrôle d'accès dans `src/lib/authz.ts`, suppression de la notion public/privé au profit des demandes d'adhésion validées, verrouillage RLS « server-only » sur toutes les tables, durcissement des codes de suppression de compte, session unique par compte (webhook Clerk `session.created`). Détail du modèle actuel dans `.claude/rules/server-architecture.md`.

  > **Incident du 22/06/2026 :** une migration de suppression de colonnes (`workshops.private`, `max_members_total`, `max_members_monthly`) a été appliquée en base alors que le code encore déployé en production les lisait → ateliers cassés en ligne. Recréées en urgence puis re-supprimées proprement une fois le code « tout privé » déployé (résolu le 25/06/2026). C'est l'origine de la règle « expand/contract » dans `CLAUDE.md` §1 — à respecter pour toute future migration destructrice.

- **24–25/06/2026 — Assainissement technique (audit).** Découpage des fichiers surdimensionnés (`ExamenTab.tsx`, `SettingsClient.tsx`) en sous-modules ; types Supabase générés (`database.types.ts`) comme source de vérité du schéma ; revalidation de cache resserrée (`revalidateWorkshop`/`revalidateDashboard`, plus jamais `revalidatePath('/', 'layout')`) ; suppression des requêtes N+1 (batch Clerk, `Promise.all`) ; nettoyage de la corbeille déplacé vers un job `pg_cron` en base plutôt qu'un appel dans un chemin de lecture chaud. Détail dans `.claude/rules/server-architecture.md`.

- **26/06–11/07/2026 — Migration i18n de l'app connectée.** Toute l'app connectée (profil, dashboard, ateliers, paramètres, générateur d'examen, jardin, session, avatar, auth) migrée de chaînes FR codées en dur vers next-intl, page par page. Migration terminée sur tout le périmètre identifié. La routine à suivre pour tout nouveau développement est dans `.claude/rules/i18n.md`.

- **11/07/2026 — Extraction de la logique métier vers `src/lib/<domaine>/`.** Toute la logique métier des server actions (membres, cycle de vie d'atelier, fichiers, générateur d'examen) déplacée vers des modules purs dans `src/lib/workshops/`, préparant une future API publique sans duplication. Les fichiers `'use server'` ne gardent que l'authz et les effets de bord Next.js (`revalidatePath`). Pattern détaillé dans `.claude/rules/server-architecture.md` — à appliquer à tout nouveau domaine dès l'écriture, pas en audit ultérieur.

- **11/07/2026 — Réorganisation de la documentation projet.** `CLAUDE.md` (alors ~1150 lignes) éclaté en : `CLAUDE.md` (racine, règles + workflow, toujours chargé), `docs/product-spec.md` (cahier des charges), `docs/backlog.md` (dette technique), `docs/changelog.md` (ce fichier), et `.claude/rules/{i18n,server-architecture,frontend-patterns}.md` (patterns de code, chargés uniquement quand les fichiers concernés sont touchés). Objectif : réduire la charge de contexte par défaut et éviter de refaire un audit de fond pour rattraper des bonnes pratiques non appliquées au fil de l'eau.

- **12/07/2026 — Fix : compte fraîchement créé inutilisable tant que `/profile` n'a pas été visité.** Le tag unique utilisateur (`publicMetadata.uniqueId` côté Clerk, ligne `user_profiles` côté Supabase) n'était généré que lazily par `ensureUniqueId()`, elle-même appelée uniquement depuis `/profile`. Un compte qui créait un atelier ou se faisait ajouter par tag avant sa première visite de `/profile` échouait — silencieusement pour `requestToJoinWorkshop` (rien n'était affiché dans `DashboardClient`), avec un message trompeur (« Non authentifié ») pour `createWorkshop`. Corrigé en faisant générer le tag directement par `syncUserProfile()` (`src/app/actions/workshops.ts`) s'il est absent, au lieu de renvoyer `null` — logique de génération extraite dans `generateUniqueUserTag()` (`src/lib/tag.ts`), partagée avec `ensureUniqueId()`. Message d'erreur générique ajouté côté `DashboardClient` pour l'échec de demande d'adhésion.

- **11/07/2026 — Audit de code clos, `AUDIT.md`/`AUDIT_2.4_design_system.md` supprimés.** Dernier point ouvert (§5.4, `middleware.ts` déprécié en Next 16) traité : fichier renommé `src/proxy.ts` (export déjà par défaut, pas de renommage de fonction), vérifié par `npm run build` (avertissement disparu, route listée `ƒ Proxy (Middleware)`) et `npm run lint` (0 erreur). Les deux fichiers d'audit ont ensuite été supprimés plutôt qu'archivés : leur contenu utile est distillé dans ce fichier/`docs/backlog.md`/`.claude/rules/`, et une vérification directe du code a montré qu'ils étaient aussi **partiellement obsolètes** — le plan d'action de `AUDIT.md` listait encore « à faire » la centralisation des couleurs et un composant `<Modal>`/`<ConfirmDialog>` partagé, alors que `src/lib/theme.ts`, `src/components/Modal.tsx` et `src/components/ConfirmDialog.tsx` existent déjà et sont utilisés. Seul le chantier Stripe (`docs/backlog.md`) reste ouvert, volontairement différé.
